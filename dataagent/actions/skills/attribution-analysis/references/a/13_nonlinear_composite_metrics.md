# A.3 非线性 / 组合指标归因

## 1. 适用范围

A 分支里的“非线性”定义：

- KPI 由多个基础指标复杂组合而来
- 无法用单一加法或简单分子分母拆解充分解释
- 需要 cross-metric / multi-metric 联合分析

**典型场景**：
- ROI、复杂评分、多因素加权指数
- 指标异常需要同时检查多个相关基础指标
- 已知或可近似得到指标关系图，但目标 KPI 不能由单一路径解释

## 2. 输入要求

- 目标 KPI 的 actual / expected，或 before / after
- 与 KPI 相关的基础指标列表
- 若可得，指标公式、依赖关系图、指标树或近似关系说明
- 若要做维度定位，需同时拿到基础指标的维度切片

## 2.1 最简定义与公式（公式已知时）

**（1）一阶泰勒（全微分）近似**

若 `Y = f(X_1, ..., X_n)` 在基准点邻域内光滑，小变化下标准多元微积分的一阶近似：

```
dY ≈ sum_i ( ∂f/∂X_i ) * dX_i
```

（一阶项；余项为高阶小量。）用于把「非线性 KPI」在**局部**线性化，再与加法型贡献叙述衔接。残差大时必须显式报告「近似失效 / 高阶项」，见第 6 节。

**（2）单因子替换（定基 / one-factor-at-a-time）**

在固定其余因子于基准水平时，仅让某一因子 `X_i` 从基期值变到当期值，比较 `Y` 的变化，可得到该因子的**边际型**贡献；**替换顺序不同一般会得到不同分解**，与 Shapley 思路互补。

**（3）Shapley 值分配（多因子路径平均）**

对「各因子对总变化 `dY` 的分配」，可将各因子视为合作者、对边际贡献按因子排序取平均，得到 **Shapley 值**（Shapley, 1953；*Contributions to the Theory of Games II*）。报告若写「Shapley 分解」，须说明是否枚举顺序、是否单独报告交互残差。

**（4）乘积型 `Y = prod_i X_i`（与 12 的衔接）**

若 KPI 可写成若干正因子乘积（如 GMV ≈ UV × CVR × AOV），常先分析 `ln(Y)`。把 `Δln(Y)` **无残差**拆到各因子上可用 **LMDI**（Ang, *Energy Policy* 2004, 32(9):1131–1139；2005, 33(7):867–871）。**若声称 LMDI**：须写明变体（LMDI I/II 等）、权重约定与可加性/残差自检。

## 3. 进入条件、退出条件、降级策略

### 进入条件

- 用 [11_additive_metrics.md](11_additive_metrics.md) 或 [12_ratio_derived_metrics.md](12_ratio_derived_metrics.md) 解释后，残差仍然较大
- 目标 KPI 本身就是复杂派生或多指标组合
- 异常不能只靠单个指标解释，必须联合看多个相关指标

### 退出条件

- 发现 KPI 其实可以被加法或简单 ratio 充分解释
- 相关指标里只有一个基础指标真正异常，其余只是跟随变化

### 降级策略

- 信息不足时，降级回 [12_ratio_derived_metrics.md](12_ratio_derived_metrics.md) 的拆分式解释
- 需要找维度候选时，叠加 [14_dimension_root_cause.md](14_dimension_root_cause.md)
- 如果只能确定“相关基础指标一起异常”，就停在候选联合解释，不强行做精细归因

## 4. 默认流程

### Step 1. 不只看异常 KPI 本身

先把目标 KPI 拆成相关基础指标集合：
- 哪些 metrics 直接参与计算
- 哪些 metrics 是其上游驱动
- 哪些 metrics 与其同步变化但只是伴生现象

若这一步拿不到最基本的 metric map，就不要直接承诺复杂分解。

### Step 2. 同时检查相关基础指标

对每个相关基础指标，先做最基础的 actual / expected 或 before / after 审计：
- 变化方向
- 变化幅度
- 是否与目标 KPI 异常方向一致
- 是否只是局部异常，还是广泛联动

这一步借鉴 CMMD / CausalRCA 类思路：复杂 KPI 不能只盯目标指标本身，必须检查相关基础指标是否共同异常。

### Step 3. 判断属于哪种非线性场景

| 场景 | 做法 |
|------|------|
| 指标公式明确，因子不多 | 用局部分解方法 |
| 已知或可近似得到指标关系图 | 沿图做联合定位 |
| 关系未知，但明确是多指标联动异常 | 作为高级路径，输出候选指标集合 |

### Step 4. 公式明确时，做局部分解

当公式明确、因子数不多时，可用：
- 泰勒一阶近似
- 有限差分 / 替换法
- Shapley 类分配

这类方法的职责是：
- 给各基础因子一个近似贡献
- 显式报告交互项 / 残差
- 若残差过大，退出精细量化

### Step 5. 关系图明确时，做跨指标联合定位

如果已有指标关系图、系统依赖图或业务上可接受的近似关系：
- 从目标 KPI 往上游追一层到两层
- 对上游指标做异常一致性检查
- 优先找“少数上游指标异常即可解释下游异常”的链路
- 输出候选指标链，而不是唯一真因

这一步是高级选项。只有在关系图相对可信时才进入。

**与 17 的边界**：本步骤的目标仍是「解释目标 KPI 为何变」，输出的是候选指标链。若问题已转为「在多个指标节点里给出根因候选**排序**」，转 [17_multivariate_metric_propagation.md](17_multivariate_metric_propagation.md)，不在本文件做图上终局排序。

### Step 6. 关系未知但明确是多指标复杂组合时

不承诺复杂 GNN、结构学习或精确因果图恢复可用。默认做法：
- 列出同步异常的关键基础指标集合
- 区分主导指标、伴生指标、未解释部分
- 若还需要定位维度组合，再叠加 [14_dimension_root_cause.md](14_dimension_root_cause.md)

高级路径（CMMD / CausalRCA 等思路）仅在关系图可信、数据质量与目标明确时进入；**不承诺** GNN、自动结构学习或完整因果图恢复可用。

## 5. 何时优先用 / 何时不要用

| 场景 | 建议 |
|------|------|
| KPI 是复杂派生或多指标复合 | 优先用本文件 |
| 用 11 或 12 后 residual 很大 | 升级到本文件 |
| 只有单一 ratio 异常，且分子分母清晰 | 先留在 12 |
| 只有单一加法指标变化，头部维度已清楚 | 留在 11 |
| 没有任何基础指标关系信息 | 谨慎，必要时降级 |

## 6. 常见失败条件 / 注意事项

| 问题 | 影响 | 处理 |
|------|------|------|
| 关系图不可信 | 联合定位可能误导 | 只做候选联合解释 |
| 基础指标定义不清 | 公式层面拆解失真 | 先补定义，再分析 |
| 多个基础指标同时轻微偏移 | 单看目标 KPI 容易误读 | 明确报告联动异常 |
| 残差过大 | 精细量化不可靠 | 降级为方向性解释 |
| 想直接跳到复杂算法 | 容易过度建模 | 先做基础审计，再升级 |

## 7. 局部调用与降级

- 若复杂 KPI 中某个局部因子本质是 ratio，可局部调用 [12_ratio_derived_metrics.md](12_ratio_derived_metrics.md)
- 若某个基础指标本质是加法型，可局部调用 [11_additive_metrics.md](11_additive_metrics.md)
- 若目标变成“哪些维度组合最可疑”，叠加 [14_dimension_root_cause.md](14_dimension_root_cause.md)
- 若信息不足，降级回 12 的拆分式解释 + 14 的维度候选定位

输出边界见 [52_output_guardrails.md](../core/52_output_guardrails.md)；须写清指标链路与未解释部分，并声明非因果。
