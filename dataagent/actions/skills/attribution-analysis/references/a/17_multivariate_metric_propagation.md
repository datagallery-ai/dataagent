# A.7 多变量指标节点传播

本文件是 A 分支的增强层。它不负责"公式怎么拆"，也不负责"哪个维度值最异常"，而负责在**一组相互可能传播的指标节点**里给出候选根因排序。叠加在 [11](11_additive_metrics.md) / [12](12_ratio_derived_metrics.md) / [13](13_nonlinear_composite_metrics.md) 之上；与 [14](14_dimension_root_cause.md) 互斥（候选对象不同），与 [15](15_adtributor.md) 不嵌套。

## 1. 何时使用

- 已确认目标指标异动窗口
- 候选对象是**多个指标节点**（不是维度切片），彼此可能存在时序前后或依赖关系
- 有可用的依赖结构：业务公式 / 流程拓扑 / 调用链 / 专家先验，或从数据学到的近似图
- 输出诉求是"候选根因指标排序"，不是因果效应估计

**不适用**：

- 目标是维度值或组合定位 → [14](14_dimension_root_cause.md)（必要时叠 [15](15_adtributor.md)）
- 目标是公式拆解 → [11](11_additive_metrics.md) / [12](12_ratio_derived_metrics.md) / [13](13_nonlinear_composite_metrics.md)
- 目标是干预增量 → C 分支

## 2. 输入要求

- 至少 2 个候选指标节点的时间序列，采样频率与窗口长度足够分辨先后
- 目标异动窗口与基线窗口；时间口径与 [02](../core/02_data_preparation.md) 对齐
- 依赖结构来源与可信度（已验证 / 部分验证 / 未验证）
- 各节点的方向语义（升=好 / 升=坏 / 待定）

## 3. 与 14 的区别（避免误用）

| 14（维度根因） | 17（多变量指标节点传播） |
|---|---|
| 候选 = 同一指标的不同 attribute-value 切片 | 候选 = 不同的指标节点 |
| 难点 = 投影重叠、不可加和 | 难点 = 节点间存在传播 / 依赖 |
| 排序依据 = 异常贡献度、覆盖度 | 排序依据 = 时间先后、依赖方向、传播权重 |

**候选既有维度又有指标节点**：先用 [11](11_additive_metrics.md) / [12](12_ratio_derived_metrics.md) / [13](13_nonlinear_composite_metrics.md) 固定 KPI 口径；维度部分走 [14](14_dimension_root_cause.md)，指标节点部分走本文件；不得合并成单一排名。

## 4. 默认流程

### Step 1. 锁定候选节点集与依赖结构

- 列出候选节点：单位、方向语义、采样频率、是否在异动窗口偏离
- 给出依赖结构来源与可信度
- 依赖结构完全不可得 → 退化到 [14](14_dimension_root_cause.md) Step 1 / 或终止本文件流程

### Step 2. 基线：时间 + 直接信号 + 机制自洽

直接复用 [14 的 Step 1.5](14_dimension_root_cause.md)（时间优先 / 信号直接性 / 机制自洽 / 输出字段）。本文件不重复其内容；在指标节点场景下，"实体"读作"指标节点"。

补充约束：

- 时间分辨率不足以区分先后 → 同窗候选并列输出
- 节点方向相反但可由同一上游解释时，须写明方向自洽假说

### Step 3. 可选增强：依赖图上的排序

依赖结构相对稳定且候选 ≥ 5 时，可在加权依赖图上做排序（例：基于入度的 PageRank 类游走），把"上游 + 影响范围"合并打分。

实现层面：可基于数据学加权 DAG 后再排序，这是其中一种参考做法。本 skill 不强制特定算法，亦不保证图上排序结果具备因果性。

### Step 4. 输出候选排序与降级条件

输出至少包含：

- 候选根因节点排序（前 3 ~ 5）
- 每个候选：偏离幅度、首次偏离时刻、依赖方向、机制自洽说明、未解释反向信号
- 依赖结构可信度等级与对结论的影响

**降级**：依赖结构可信度低 / 时间分辨率不足 / 候选 < 2 → 只能写"候选解释"，不得写"候选根因"。

**升级**：结论要用于 ON/OFF 干预或资源调配 → 必须升级 C 分支做因果校准。

## 5. Guardrails

本节只补充本方法的专项风险；通用输出边界见 [52](../core/52_output_guardrails.md)。

- 图上排序的高分节点 ≠ 因果原因；仅是"在该依赖结构下最可能的上游候选"
- 不同依赖结构来源给出不同排序时，输出多版排名并说明分歧
- 学得图对窗口长度与平稳性敏感；窗口外推须重新估计
- 单纯相关 / 同步异常排序不得作为最终结论（[14 Step 1.5](14_dimension_root_cause.md) 禁令同样适用）

## 6. 与 11 / 12 / 13 / 14 / 15 的关系

- 11 / 12 / 13 决定"目标 KPI 公式怎么拆"
- 14 决定"在 attribute-value 切片里集中在哪"
- 15 在有 forecast 时做维度首轮 narrowing
- 本文件在"候选是指标节点 + 有传播 / 依赖"时使用，与 14 互斥，与 15 不嵌套

输出须标明排序所依赖的结构来源与可信度；结构化输出边界见 [52](../core/52_output_guardrails.md)。
