# A.6 横截面群体对比根因

本文件用于回答"**为什么 X 群体的指标比 Y 群体高/低**"——同一时点、两个或多个共时群体之间的指标差异分解。

## 1. 何时使用 / 何时不要用

**使用本文件**：
- "VIP 用户的转化率为什么比普通用户高 8pp"
- "北区门店人均产出比南区低 12%，差在哪"
- "新客 7 日留存比老客低，是因为渠道结构还是行为差异"

**不要用本文件**：
- 同一群体随时间变化 → 走 [10](10_metric_anomaly_overview.md) → 11/12/13 + [14](14_dimension_root_cause.md)
- 想说"X 群体之所以高，是因为我们做了 Y" → **必须升级到 C 分支**（见 [01 的 5a](../core/01_problem_classification.md)）；本文件只解释相关性结构差异，不给因果结论
- 找异常数据点（不是群体） → [14](14_dimension_root_cause.md) 的维度打分

## 2. 与 14 的关键区别

| 14（维度根因） | 16（群体对比） |
|---|---|
| 比较对象：同一群体在两个时点 | 比较对象：同一时点的两个群体 |
| 时间维度上的"变了多少" | 群体维度上的"差了多少" |
| 关注异常 / 偏离基线 | 关注组间结构差异 |
| 适用：异动归因、forecast 偏差 | 适用：分群报表、群体画像、AB 群对照 |

混用风险：把 14 套到群体对比，会把"X 群体内部最异常的维度"误当成"X vs Y 差异的来源"。两者经常不一样。

## 2.1 Blinder–Oaxaca（组间均值差）

对两组 A、B 分别估计 `beta_A`、`beta_B`（Oaxaca 1973；Blinder 1973）：

```
mean(Y_A) - mean(Y_B)
  = beta_A' * (mean(X_A) - mean(X_B)) + mean(X_B)' * (beta_A - beta_B)
```

第一项常称 **explained / endowments**（特征差异按某一组系数计价），第二项为 **unexplained / coefficients**（系数差异；解释需谨慎）。`Y` 既可为水平也可为变换后水平（如对数工资）；与第 3 节 Step 2 表格中的「结构 / 系数」叙述一致。

**与 Kitagawa 率分解**：在二元结果、互斥类别协变量且用线性概率模型等限定下，分层 Kitagawa 率差与 Blinder–Oaxaca 可对齐；一般设定下 **Blinder–Oaxaca 更一般**。率的分层对称分解式见 [12 的 3.1 节](12_ratio_derived_metrics.md)。

## 3. 标准做法

### Step 1. 锁定指标公式与可比口径

- 指标是加法型 / 比率型 / 复合型（沿用 [11](11_additive_metrics.md) / [12](12_ratio_derived_metrics.md) / [13](13_nonlinear_composite_metrics.md) 的判定）
- 群体定义、统计口径、时间窗、分母对齐——任一不一致 → 先口径对齐，禁止直接比对
- 群体之间的样本量差异要记录；单边样本过小（如 n < 30）退化为"探索性观察"

### Step 2. 选差异分解方法

| 指标类型 | 默认方法 | 说明 |
|---|---|---|
| 加法型（总量、计数） | 按子项加法分解：`gap = Σ (x_i^X - x_i^Y)` | 直接逐项相减，叠加 11 的可加约束 |
| 比率型（CR、CTR、ARPU） | **Oaxaca-Blinder 分解**（结构效应 + 系数效应） 或 标准化（standardization） | 区分"群体构成不同"vs"同结构内表现不同" |
| 复合 / 非线性 | 先固定其他因子做 Shapley / KernelSHAP on group label | 必要时按 [13](13_nonlinear_composite_metrics.md) 的非线性处理 |

**Oaxaca-Blinder 的最小形式**（比率型最常用）：

```
Y_X - Y_Y =  (X_X - X_Y) · β_Y    +    X_X · (β_X - β_Y)
              \________/             \________________/
              结构 / 构成效应          系数 / 行为效应
```

- 结构效应：因为两个群体在协变量分布上不同（VIP 主要走 App，普通用户走 H5）
- 系数效应：因为同样的协变量在两个群体身上效应不同（VIP 的 App 转化率本身就更高）
- 报告必须**两者都给出**，不要只报其中一项

### Step 3. 解释强度分级

按 [52 §4](../core/52_output_guardrails.md) 的结论强度分级，但本文件**不允许**出现 evidence_level=`增量估计` 之外的因果性措辞：

- ✅ "VIP 与普通用户转化率差 8pp，其中 5pp 来自渠道结构差异，3pp 来自同渠道内行为差异"
- ❌ "VIP 渠道结构导致了 5pp 的转化率提升"（这是因果断言，需走 C）

### Step 4. 控制混淆 / 自选择风险

群体本身往往是**自选择**结果（VIP 不是随机的）。报告必须显式声明：

- 哪些观察到的协变量做了对齐 / 标准化
- 哪些未观察到的混淆可能驱动差异（"VIP 用户本身偏好就不同"）
- 是否需要升级到 C 分支做 matching / IPW / DML（升级条件见 [01 的 5a](../core/01_problem_classification.md)）

## 4. 反模式

| 反模式 | 后果 | 应做什么 |
|---|---|---|
| 直接说"X 群体高是因为 X 群体的人做了 Y" | 把相关性当因果，下游决策被误导 | 用 Oaxaca 分两层；想下因果结论必须走 C |
| 不做口径对齐就跨群体比 | 比的是不同的东西 | Step 1 先对齐分母 / 时间窗 / 定义 |
| 用 14 的维度打分代替 Oaxaca | 找到的是 X 群体内部异常项，不是 X vs Y 的差异源 | 群体对比走 16，时序异动走 14 |
| 群体内样本极不均衡却报点估计 | 小样本侧噪声主导 | 小样本侧标 `insufficient_n=true`，给区间不给点估计 |
| 只报结构效应或只报系数效应 | 解释不完整，决策方向相反 | 两层都报 |

## 5. Guardrails

- 本文件输出按调用方契约收口，边界见 [52](../core/52_output_guardrails.md)；`question_type` 填 `群体差异分解`
- `analysis_object` 必须显式说明对比的两个群体定义
- 结论必须标注"相关性差异分解，不构成因果证据"（落到 evidence_level / 限制字段）——除非已升级到 C
- 升级路径见 [01 的 5a](../core/01_problem_classification.md)
