# 数据准备与口径固定

本文件在确定分支后、方法选型前使用。所有分支共用数据质量预检流程，但数据形态因分支而异。数据口径问题比方法选型问题更常见、更致命。

## 1. 数据质量预检（所有分支共用）

### 通用检查清单

| # | 检查项 | 怎么查 | 不通过时怎么办 |
|---|--------|--------|------------|
| 1 | 口径一致性：两期/两组的定义、筛选条件是否一致 | 对比 SQL/定义文档 | 修正或标注不可比 |
| 2 | 数据完整性：缺失日期、补数、回填、延迟回传 | count by date + null 率 | 记录缺失范围，评估影响 |
| 3 | 采集变更：埋点改版、SDK 升级、平台策略变化 | 查 changelog | 变更前后分开分析 |
| 4 | 异常值与噪声：极端值、隐私噪声 | 分布检查 + IQR | 标记或裁剪 |
| 5 | 维度完整性：枚举值是否在两期间新增/消失 | distinct values 对比 | 对齐枚举或标注 |

### B/C 分支额外检查

| # | 检查项 | 说明 |
|---|--------|------|
| 6 | 身份与去重：跨设备/跨域/跨平台导致路径断裂 | 评估统一 ID 覆盖率 |
| 7 | 分组完整性：treatment/control 分组逻辑与曝光资格 | 验证随机化质量 |

### 问题处理规则

| 问题严重度 | 处理 |
|----------|------|
| 可修正 | 记录修正方式，继续 |
| 不可修正但影响有限 | 记录风险，降低结论强度 |
| 不可修正且影响重大 | **终止精确归因**，输出已知信息 + 数据问题说明 |

**关键判断**：如果口径变化 > 真实业务变化（即数据问题比业务信号大），必须终止归因，否则会把"对齐误差"当成"渠道效应"。

## 2. 口径对齐与基准定义（按分支）

| 分支 | 需要对齐的口径 |
|------|------------|
| A 指标异动 | 对比期选择：同比/环比/预测值/目标值 |
| B0 通用功劳分配 | 结果总量、候选因素集合、分配单位、加和/归一化规则 |
| B1 MTA | 归因窗口、转化定义、去重规则、曝光 vs 点击 |
| B2 MMM | KPI 定义、渠道层级、控制变量、adstock/饱和假设 |
| C 因果 | 反事实定义、识别假设、调整集 |

## 3. A 分支的基准期选择规则

### 优先级

| 优先级 | 场景 | 基准设定方式 | 检查要点 |
|--------|------|-----------|---------|
| 1 | 已有 expected/forecast 类基准（监控带、预测区间、SLO baseline 等） | actual vs expected/forecast | 基线/预测是否近期重训或口径变更 |
| 2 | 日常经营复盘 | 同比或环比（优先同比消除周期性） | 无口径变更、无节假日偏差 |
| 3 | 目标差距分析 | actual vs target | 说明是 gap 分析而非自然异常 |
| 4 | 预算/计划对比 | actual vs plan | plan 口径一致 |

### 基准期检查（必做）

- 两期口径一致（筛选条件、采集方式）
- 排除不可比因素（节假日、大促、采集变更）
- 多基准交叉：对同一异常使用 ≥2 个基准（如同时看同比和环比），判断是否一致
- 基期数据量过小 → 降低结论置信度

### 常见错误

- 用大促期间数据作为日常复盘的基准
- 基准期与当期跨越了采集/口径变更节点
- 用年度目标值作基准解释周维度的波动

### 3.1 异常子窗口对齐（A 分支通用，因果链根因必查）

当对比基准是“正常 vs 异常”而非“同比 / 环比”时（典型：故障复盘、突发异动、告警触发后的回溯），所有打分、相关系数、Z-score 都应在**统一锁定的异常子窗口**内计算，而不是在全样本窗口里。

**最小要求**：

| 字段 | 含义 | 选取原则 |
|---|---|---|
| `event_window` | 主异常窗口 | 包含异常发生 → 缓解的时段；若有显式事件 / 变更 / 触发时间，从该时间点开始 |
| `baseline_window` | 同长度的正常对照窗口 | 与 event 同口径、同周期位（同 weekday / 同小时段），尽量靠近 event 之前 |
| `propagation_buffer` | 传播缓冲 | event 起点向后扩展若干个观测周期，覆盖后继传播延迟 |

**硬规则**：

1. 全样本窗口内的相关性 / Z-score 只能作为背景参考，不得直接作为根因排序依据
2. 任何被纳入根因排序的指标，必须报告“在 event_window 内的异常幅度”和“在 baseline_window 内的同口径基线水平”两个数
3. 若 event 的起止时间无法确定，必须在输出中显式标注“窗口未对齐，结论降级为统计集中”
4. 若候选实体之间存在因果链 / 依赖关系（参见 [01 的 3.1 节](01_problem_classification.md)），还要额外记录每个候选在 event_window 内的“首次偏离时间戳”，供 [14 的 Step 1.5](../a/14_dimension_root_cause.md) 做时间优先性判断

## 4. 归因事实表形态（按分支）

| 分支 | 数据形态 | 最低要求 |
|------|--------|--------|
| A | 指标值表 + 维度切片表（两期） | 指标值 + 至少一个维度 + 两期数据 |
| B0 通用功劳分配 | 结果表 + 候选因素参与证据表 | 已发生结果 + 候选因素 + 分配单位 + 可追溯参与证据 |
| B1 MTA | user_id / journey_id 路径表 | 用户标识 + 触点序列 + 转化标签 + 时间 |
| B2 MMM | geo × time 面板 | 媒体投入 + KPI + 控制变量 + ≥52周 |
| C | treatment + outcome + covariates | treatment 标识 + 结果 + pre-treatment 协变量 |

## 5. B0 通用功劳分配的事实表构建规则

### 数据组织

```text
result_id, result_value, result_time_window, allocation_unit,
entity_type, entity_id, participation_signal, participation_value(optional),
allocation_rule(optional), evidence_source, eligibility_flag
```

### 构建要求

- 一行应能追溯到“某个候选因素为什么有资格分享某个结果”的证据。
- 若同一结果可由多个因素分享，必须保留 `result_id`，用于检查同一结果下的分配是否加总到 100% 或总价值。
- 若候选因素只参与排序、不要求分摊总量，应显式标注 `allocation_unit=ranking_only` 或等价说明。
- 若使用活动量、成本、人天、使用次数等作为分配依据，需保留原始 `participation_value`，不要只保留归一化后的份额。
- eligibility 规则必须在建表前固定；不得看到结果后临时剔除低贡献因素。

## 6. B1 MTA 的路径构建规则

### 会话切分（Sessionization）

- 基于时间间隔：两个相邻触点间超过 30 分钟则切分
- 基于转化事件：每次转化作为路径终点
- 基于日期切割：按自然日切分

### 触点去重规则

| 场景 | 处理方式 |
|------|---------|
| 同渠道同会话多次触达 | 合并为 1 次（保留首个或末个时间戳） |
| 曝光与点击并存 | 点击优先 |
| 重复展示广告 | 设频次上限（如每天最多计 3 次） |

### 归因窗截断（Lookback Window）

- 转化前 N 天内的触点纳入路径，更早排除
- 典型窗口：7天（短周期商品）、28天（高客单价）、90天（SaaS）
- 窗口越长路径越长，Shapley 组合空间爆炸

### 未转化路径处理

- **必须保留未转化路径**——只保留转化路径会导致选择偏差
- Markov 模型需要未转化路径来估计吸收概率
- 标记 `converted = 0/1` 列，不要只筛选 `converted = 1`

## 7. B2 MMM 的面板构建规则

### 数据组织

```
index: [geo_id, date]          # geo × time 面板
KPI列: revenue / conversions / signups
媒体投入列: spend_search, spend_social, spend_display, spend_tv, ...
控制变量: price, promotion_flag, competitor_spend, gdp_index, ...
季节/趋势: week_of_year, holiday_flag, ...
```

### Adstock 变换
- 几何衰减：`adstock_t = spend_t + λ * adstock_{t-1}`，λ ∈ [0, 1]
- 典型衰减率：搜索广告 λ≈0.1-0.3（快衰减）、品牌广告 λ≈0.7-0.9（慢衰减）

### 饱和变换
- Hill 函数：`y = x^α / (x^α + K^α)`

## 8. C 分支的因果分析表构建规则

### 核心字段

| 字段 | 含义 | 关键要求 |
|------|------|---------|
| `treatment` | 干预变量 | 必须明确干预定义 |
| `outcome` | 结果变量 | 必须在 treatment 之后观测 |
| `covariates` | 协变量 | 只包含 treatment 前的变量 |
| `assignment` | 分组分配 | 实验的随机分配 |
| `eligibility` | 曝光资格 | 有资格被分配的人群标志 |

### 时间关系要求

```
covariates(t0) → treatment(t1) → outcome(t2)
必须确保 t0 < t1 < t2
```

将 treatment 后的行为当协变量 = 中间变量偏差，这是最常见的构造错误。

## 9. 隐私约束下的数据准备（此处基于多文归纳整理）

在去第三方 cookie 等环境下：

- 浏览器侧归因 API 提供 event-level 和 aggregatable 两种报告
- 聚合报告引入噪声、贡献预算、报告延迟
- **经验法则**：先在"能支持的最粗粒度"上把 measurement 跑通，接受噪声与预算约束带来的方差，再逐步细化维度
- 把"口径版本化"与"噪声模拟/回放"加入评估体系
