# Phase 7：结构化输出护栏

本文件只规定输出契约优先级、结构化结果一致性和结论强度边界。它不定义 JSON、schema、字段名、字段顺序或字段是否必填；这些由调用方 prompt、评测脚本或用户指令定义。

## 1. 契约优先

调用方给出 JSON schema、字段集合、机器可读块或其他结构化契约时：

1. 只按调用方契约输出。
2. 不额外添加正文、Markdown、代码块、收口句或过程说明，除非契约要求。
3. 计划、todo、工具回执、日志摘要不算最终交付。
4. 主估计锁定后即可交付；互核、refutation、稳健性、敏感性分析未完成时，只降低结论强度，不阻塞结构化输出。
5. skill 只负责口径、识别策略、主方法和 sanity 逻辑，不为 A/B/C 分支另设字段级契约。
6. 各 reference 子文件里的「报告 / 正文 / 段落 / 措辞 / 声明 / 收口句 / 交付收口」等说法，指的是结论口径、结论强度和必须核对的量，**不是要求生成文字报告**。调用方要求结构化输出时，这些信息一律落到对应字段或 `evidence_level`，不额外成段。

## 2. 结构化一致性

交付前核对：

```text
output_consistency_check:
- 主分支 == 主方法口径
- 归因口径 == 结论字段含义
- 归因对象 == 主结果对象
- 主方法 == 主估计 / 主要证据
- 结论强度 == 不确定性与校准状态
result: pass | fail
```

若不一致，以主方法和主估计为准修正结构化字段；不要为了补齐字段而伪造口径。

## 3. 口径边界

| 口径 | 可输出 | 禁止 |
|------|--------|------|
| A 指标异动 | 统计集中、候选解释、候选根因 | 写成因果增量 |
| B 功劳归因 | credit、contribution、模型依赖贡献 | 写成因果 ROI / 增量贡献，除非已接 C 校准 |
| C 因果归因 | effect、standard error、estimand、识别策略 | 策略和 estimand 错配，如 sharp RDD 写 LATE |

## 4. 结论强度

| 强度 | 条件 | 适用输出 |
|------|------|----------|
| 统计集中 | 只有拆解，无机制证据 | A |
| 候选解释 | 拆解 + 方向合理 + 部分佐证 | A |
| 候选根因 | 拆解 + 机制/事件/变更证据 | A |
| 模型依赖贡献 | 有功劳分配模型，但无增量校准 | B |
| 增量估计 | 有实验/准实验识别 | C |
| 降级待复核 | 数据质量、识别假设、sanity 或 refutation 存疑 | A/B/C |

独立对照、refutation、敏感性分析用于提升或降低结论强度；结构化契约不要求这些字段时，不额外输出。

## 5. 停止量化条件

出现以下情况时，优先输出契约允许的空值、降级标记或错误说明，不强行编造数值：

| 条件 | 处理 |
|------|------|
| 数据缺失或关键列不存在 | 不生成对应数值 |
| 口径变化大于业务变化 | 降级为数据问题 |
| 处理和结果时间顺序不成立 | 不输出因果估计 |
| C 分支识别策略无法成立 | 降级为候选解释或待复核 |
| 主估计量级超出结果变量合理范围 | 降级为待复核 |
