# Phase 5：推断与不确定性

本文件在方法执行完成后使用，用于诊断补充、高风险决策和结论强度校准。调用方只要求结构化输出时，按契约先交付主估计，未完成的不确定性检查只降低结论强度。

## 1. 核心原则

归因结果若用于决策，区间估计比点估计更关键。"渠道A贡献60%"远不如"渠道A贡献60% [45%-75%]"有用。

## 2. 各分支的不确定性要求

| 要求项 | A 指标异动 | B Contribution Attribution（功劳归因） | C 因果 |
|------|---------|---------|-------|
| 置信/后验区间 | 可选 | 模型估计建议；MMM 可作诊断补充 | C 可作诊断补充 |
| 残差与加和校验 | 必须 | B0/B1 建议 | — |
| 敏感性分析 | — | 规则分配与 MMM 建议 | C 可作诊断补充 |
| 假设检验/refutation | — | — | C 可作诊断补充 |

## 3. A 分支：残差与加和校验

- 各维度项贡献之和**必须等于**总变化量（加法型）
- 比率/非线性指标：检查残差/交互项大小
- 残差 > 主效应 20% → 标注"近似拆解"
- 不同维度的贡献不可直接加总（是不同投影）

## 4. B 分支：不确定性（功劳归因；B0/MTA/MMM）

### B0 通用功劳分配
- 规则分配本身**无统计不确定性**，但可检查规则敏感性（例如权重、窗口、候选因素集合变化后排序是否稳定）。
- 模型化贡献可用 bootstrap、交叉验证或参数区间给出范围。

### MTA
- 规则归因**无不确定性**（它是定义，不是估计）
- 数据驱动 MTA 可通过 bootstrap 给出区间

### MMM
- 贝叶斯框架（Meridian/PyMC-Marketing）天然给出后验区间
- **必须输出渠道贡献的后验分布**
- 敏感性分析：改变 adstock 衰减率 ±20%，看渠道贡献排序是否变

## 5. C 分支：假设检验与 refutation（用于结论强度）

| 检验 | 做什么 | 预期 | 不通过时 |
|------|--------|------|---------|
| Placebo | 用假 treatment | 效应为零 | 模型有问题 |
| 子样本稳定性 | 在子集上复现 | 结果一致 | 效应不稳健 |
| 未观测混杂敏感度 | E-value | 需要多强混杂 | 混杂可能消除效应 |
| 随机噪声注入 | 随机变量做 treatment | 效应不显著 | 模型过拟合 |

**工具**：DoWhy refutation API

## 6. 不确定性量化方法

### Bootstrap
- 对数据重采样 B 次，每次重跑分析
- 适用：A 分支贡献度、B1 MTA 的数据驱动归因
- 类型：非参数 / 分层 / block（时序）

### 贝叶斯后验
- MMM 框架天然输出
- credible interval 比点估计更有信息量

### Delta 法
- 估计量可表示为参数的光滑函数时，用 Taylor 推导方差
- 适用：A 分支比率指标的分解不确定性

### 敏感性分析
- 系统性改变关键假设/参数，观察结论是否稳定
- 结论随假设大幅变化 → 降低结论强度

## 7. 结论强度与不确定性的映射

| 不确定性水平 | 结论强度 |
|------------|---------|
| 区间窄、假设通过验证 | 可用确信表述 |
| 区间宽但方向一致 | 输出方向 + 量级范围 |
| 假设部分不成立 | 降级为候选解释 |
| 残差/交互明显、假设脆弱 | 只输出定性方向 |
