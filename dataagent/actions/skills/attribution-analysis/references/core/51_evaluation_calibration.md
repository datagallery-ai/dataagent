# Phase 6：评估与校准

本文件覆盖"如何验证归因结果可信、可复现、可行动"。

## 1. 核心原则：三角校验（triangulation）

不依赖单一方法下结论。多种方法方向一致时可提高结论强度。

```
方法 1（如规则功劳分配 / MTA） ←→ 方法 2（如 MMM / 替代模型） ←→ 方法 3（如实验）
  方向一致？→ 提高结论强度
  不一致？  → 排查口径/假设/数据差异
```

## 2. 各分支的评估方法

### A 指标异动
- **加和校验**：各贡献项之和 = 总变化
- **头部覆盖度**：前 N 项解释了多少
- **可行动性检查**：结论能否转化为行动

### B Contribution Attribution（功劳归因）
- **与实验增量对齐**：用 lift test / geo-test 或等价对照校准 B0/B1/B2 的尺度（详见 23）
- **回测与 holdout**：时间切分回测，或在可保留样本上检验贡献排序稳定性
- **一致性检查**：贡献方向、规模、边际递减或资源约束是否符合业务约束
- **交叉方法对比**：规则分配、MTA、MMM、Shapley/回归等方法的贡献排序与量级是否一致

### C 因果归因
- **有真值时**：ATE error, PEHE, coverage
- **无真值时**：refutation + 敏感性分析
- **决策价值**：policy value / expected response
- **不能用普通 AUC 评估 uplift**

## 3. 评估与"用途"绑定

评估指标必须和归因用途对齐：

| 容易犯的错 | 为什么错 |
|----------|---------|
| 用"高忠实度"证明"好的预算分配" | 解释忠实度 ≠ 决策质量 |
| 用"符合直觉"证明"因果增量" | 直觉 ≠ 因果 |
| 用"加和校验通过"证明"找到了根因" | 数学分解正确 ≠ 根因正确 |
| 用"预测准"证明"因果对" | 预测好可能利用了混杂 |

## 4. 评价指标速查表

| 类别 | 代表指标 | 适用领域 | 主要限制 |
|------|---------|---------|---------|
| 拟合/预测 | MAPE/RMSE/R² | MMM、转化模型 | 预测好不代表因果对 |
| 因果增量 | ATE error, PEHE | 因果归因 | PEHE 需真值/半合成 |
| 决策/排序 | AUUC/Qini/Policy value | uplift | 需无偏估计或 RCT |
| 工程质量 | 可复现、口径完整、假设披露 | 全部 | 评分主观 |

## 5. 上线监控与治理

### 数据漂移监控

| 监控项 | 方法 | 触发动作 |
|--------|------|---------|
| 指标分布偏移 | PSI / KS 检验 | PSI > 0.2 → 重分析 |
| 维度枚举变化 | 新增/消失值检测 | 更新维度映射 |
| 模型预测偏移 | MAPE 监控 | 超阈值 → 重训 |
| 归因结果异常 | 渠道贡献 vs 历史 | 偏离 → 人工复核 |

### 口径变更管理
- 所有口径变更必须记录版本日志
- 变更前后的归因结果做 A/B 对比
- 变更后首次结果标注"口径变更后"

### 版本化
- 输入（数据版本、参数、模型版本）可追溯
- 每次分析关联唯一 run_id

### 复盘
- 每月/季度复盘归因结果与实际决策的对齐度
- 上次建议是否执行 + 效果是否符合预期
- 不符合 → 排查原因（数据/方法/市场变化）
