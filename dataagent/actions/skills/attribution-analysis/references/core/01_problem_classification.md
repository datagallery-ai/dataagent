# 问题分类与路由

本文件用于判断归因问题属于哪一大类，决定进入哪条方法路径。分类错误会导致用错整套方法体系，因此本步必须在方法选型前完成。

## 1. 一级分类决策树

```
归因问题
│
├─ Q1: 核心诉求是「变化/异常/群体差异从哪来」（**A：指标变化与差异解释**），还是「**已发生结果下的功劳分配或因果增量**」（先不分 B/C，继续 Q2）？
│  ├─ 变化与差异解释 → 【A. 指标异动归因】→ 10
│  │   └─ A 内部再分：时序变化 → 11/12/13(+14/15)；同时点群体对比 → 16
│  └─ 功劳分配或因果增量 → 继续 Q2
│
└─ Q2: 要的是 **C：因果增量**（须因果识别与估计），还是 **B：Contribution Attribution / 功劳归因**（在既定结果下做 credit allocation 即可）？
   ├─ 因果增量 → 【C. 因果归因】→ 30
   └─ 功劳归因 → 继续 Q3
      │
      └─ Q3: 候选贡献因素与数据形态是什么？
         ├─ 用户级触点 / 行为路径 → 【B1. MTA / DDA — 路径级功劳】→ 21
         ├─ 聚合级投入-结果面板（渠道×周/月、geo×time 等） → 【B2. MMM / 聚合贡献】→ 22
         └─ 其他资源 / 功能 / 团队 / 流程节点等贡献分配 → 【B0. 通用功劳分配】→ 20a
```

## 2. 判别信号详解

### 如何判断是 A 还是 B

| 信号 | A（指标异动） | B（Contribution Attribution / 功劳归因） |
|------|------------|-------------|
| 问法 | "为什么下降了""变化来自哪里" | "谁拿 credit""各触点/渠道/资源/功能贡献多少""预算/资源怎么分（基于功劳）" |
| 数据需求 | 指标值 + 维度切片 + 对比基准 | 已发生结果 + 候选贡献因素 + 分配规则/路径/面板/模型输入 |
| 分析对象 | 指标本身的变化 | 对既定结果有参与关系的因素集合 |
| 核心输出 | 哪些维度项解释了变化 | 功劳份额、贡献金额/数量、贡献排序、可选预算/资源建议 |

**常见误判**："GMV 下降了，各渠道贡献多少"——若无功劳分配语义、只是**两期按渠道切片分解变化**，属 **A**，不是 **B**。仅当问题明确是在**已发生结果下分配 credit / contribution**（如用户路径功劳、渠道贡献、资源贡献、功能贡献、聚合层边际贡献）时才是 **B**。

### 如何判断是 B 还是 C

| 信号 | B（功劳归因） | C（因果增量） |
|------|------------|------------|
| 是否要求因果解释 | 不要求，相关性分配即可 | 要求，需要因果证据 |
| 是否有 treatment/control 概念 | 无 | 有 |
| 是否涉及反事实 | 不涉及 | "如果没有这个干预会怎样" |
| 决策风险 | 中（参考性分配） | 高（用于因果判断和策略决策） |

## 3. A 分支内部的二级分类

当进入 A（指标异动归因）后，需进一步判断指标的公式结构：

| 判别方式 | 子类型 | 主参考文件 | 判别信号 |
|---------|--------|---------|--------|
| Y = Σy_i，各子项加起来等于总值 | 加法型 | 11 | GMV = 各渠道GMV之和；订单量 = 各品类订单之和 |
| Y = A/B，分子分母同时变化 | 比率/派生型 | 12 | 转化率 = 转化数/访问数；ARPU = 收入/用户数 |
| Y = f(X1,...,Xn)，有交互/非线性 | 非线性/组合型 | 13 | ROI = (Conv × ARPC) / Cost |
| "哪个维度/维度项最异常"（沿时间下钻） | 叠加维度根因定位 | 14 (+15) | 在任何子类型之上叠加 |
| "为什么 X 群体比 Y 群体高/低"（同时点对比） | 横截面群体对比 | 16 | 不是时序异动，是两个共时群体的指标差异 |

**判别步骤**：
1. 写出指标的计算公式
2. 如果是纯加总 → 11
3. 如果是 A/B 的比值 → 12
4. 如果涉及多因子乘/除/幂 → 13
5. 不确定 → 先当加法型处理（最简单），发现不可加时再切换

**常见误判**：
- 把转化率当加法指标拆（转化率 ≠ 各渠道转化率之和）→ 应进 12
- 把「渠道贡献」当成路径功劳：无路径、仅为维度拆解 → **A**；有路径或功劳/边际建模 → **B**

### 3.1 候选实体之间存在因果传播时的特殊处理

**触发信号**：候选对象不是相互独立的维度值，而是**沿因果链 / 依赖关系**相互影响的实体（任何形式的“前驱→后继”结构：上下游流程、母子模块、源→宿、漏斗上下层、依赖图节点等）。此时一次异常会沿链传播，把多个实体同时拉偏。

**与普通维度根因的区别**：

| 普通维度根因（14 默认） | 因果链根因 |
|---|---|
| 维度值之间相互独立、可加和 | 实体之间有因果传播，不可简单加和 |
| 头部维度 = 解释力最大的项 | 根因实体 ≠ 解释力 / 异常幅度最大的实体（往往前驱小幅异常引起后继大幅异常） |
| Z-score / 贡献度排序即可 | 必须叠加“时间优先 + 传播方向 + 直接信号”过滤 |

**硬规则**：识别为本子类后，14 的单纯打分排序**不能直接作为最终结论**，必须叠加 [14 的 Step 1.5](../a/14_dimension_root_cause.md) 的过滤层（时间优先、信号直接性、机制假说、层级化输出）。

**候选对象再分流**：

- 候选 = 同一指标在不同 attribute-value 上的切片（即便切片之间有上下游 / 父子关系）→ 留在 [14](../a/14_dimension_root_cause.md)
- 候选 = **多个不同的指标节点**（同一对象上的不同指标时序）且节点间有传播 / 依赖结构 → 转 [17](../a/17_multivariate_metric_propagation.md)

**保守默认**：因果链 / 依赖关系未知或不可得时，输出应明确标注“因果链基于 X 推断，未经独立验证”，并给出候选集合而非唯一根因。

## 4. B 分支内部的二级分类（Contribution Attribution / 功劳归因）

| 信号 | 子类型 | 主参考 | 选择理由 |
|------|--------|--------|--------|
| 有用户/会话级触点轨迹，要把功劳分到触点、创意、关键词、步骤等 | MTA / DDA | 21 | 可做路径级 credit allocation |
| 只有渠道/区域×时段的投入与结果汇总，要做整盘预算、资源配比或边际贡献 | MMM / 聚合贡献 | 22 | 聚合层建模，常含多触达渠道、线下或外部控制变量 |
| 候选因素是资源、功能、团队、流程节点等，目标是对已发生结果分摊功劳 | 通用功劳分配 | 20a | 先固定结果、候选因素、分配规则和可加性/交互假设 |
| 需要因果标尺校准 B 分支功劳或要声称增量价值 | 增量测试 | 23 | 提供因果校准 |

**B 内部选择决策**：
- 核心问题是**战术层渠道/素材/触点优化**且有较完整的用户级触点数据 → MTA
- 要回答**策略层跨渠道预算与长期效应**、存在隐私/信号缺失或线下渠道 → MMM
- 要回答**非营销场景的资源/功能/流程贡献**，且没有用户触点路径或媒体投入面板 → 进入 [20a](../b/20a_general_contribution_allocation.md)，选规则分配、活动量分摊、结果映射、Shapley/回归等合适方法
- 两者都有数据 → 并行，用增量测试做三角校验
- Marketing Attribution 是 B 的重要应用；MTA、DDA、MMM 本质上都属于 B 的营销形态，但 Marketing Attribution 领域还可能包含 A/C/预算优化等问题，不能与 B 画等号

## 5. 分支叠加规则

一个任务可以跨分支，但需区分主次：

| 任务描述 | 主分支 | 辅分支 | 说明 |
|---------|--------|--------|------|
| "GMV 下降了，各渠道贡献多少" | A | — | 渠道维度拆解，**不是 B（功劳归因）**，除非明确要对既定 GMV 分配 credit |
| "各渠道/各触点/各资源功劳分配，需要因果证据" | B | C | 用增量测试或其他因果识别校准 B 结论 |
| "新策略上线后 GMV 变化的原因" | A | C | 先做维度拆解，再看因果增量 |
| "VIP vs 普通用户为什么转化率不同" | A | — | 横截面群体对比，进 16 |

写明"主分支"决定方法选型顺序，"辅分支"在主分支完成后叠加。

## 5a. 分支升级触发条件

判完主分支后，遇到下面任一信号，必须**升级到上一档强度**，不能停留在原分支结论上：

| 当前分支 | 升级触发信号 | 升级到 | 操作 |
|---|---|---|---|
| A（异动 / 对比） | 用户要求"做点什么"（调投放、改产品、动预算、修模型）而不只是"了解一下" | A + C | A 负责定位嫌疑维度，C 负责估计干预效应；A 的结论降级为"相关性候选解释" |
| A（沿因果链 / 依赖关系的实体） | 候选实体之间存在前驱→后继传播（见第 3.1 节） | A + 14 的 Step 1.5 过滤层 | 单纯打分排序不再作为最终结论 |
| B（功劳归因，尤其 21/22） | 结论用于"加预算 / 砍预算 / iROAS 报告 / 声称增量价值" | B + C，走 23 做增量校准 | B 数字定为"相关性 credit / 模型依赖贡献"，因果数字以增量测试或 C 分支识别为准 |
| B（22 MMM） | 用于战略预算决策、跨年度规划，或参数对预算敏感 | B + C，走 23 做增量校准 | 至少一个主渠道或关键资源项要有增量测试校准 priors |
| C 任何子路径 | 主估计依赖单一识别策略且决策可逆性低 | C 内部三角校验 | 至少两种识别策略方向一致才提升结论强度 |

**保守默认**：未升级时，结论必须显式标注口径上限（"本结论为相关性 / 维度集中度，不构成因果证据"）。

## 6. 类型不明时的保守策略

1. 先让用户填写对齐句式（00）
2. 如果仍不清楚，**优先按 A（指标异动）处理**——A 是最保守、假设最弱的分支
3. 不要在口径不明时就开始选方法
4. 如果分析过程中发现类型判断错误 → 回退到本步重新分类，不要硬做
